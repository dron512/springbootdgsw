
스프링 프레임워크를 이용해서 아래 기능을 만족하는 게시판을 구현하세요.

글 쓰기 페이지 : /write
    - 입력 항목
        - 제목 : 최대 100자(영문자 기준)
        - 내용 : 최대 1000자
        - 글쓴이 : 최대 50자
    - '저장' 버튼을 클릭하면 사용자가 입력한 글이 등록되고, 글 목록 페이지로 전환합니다. 글 목록 페이지

글 수정 페이지 : /update
    - 입력 항목
        - 제목 : 최대 100자(영문자 기준)
        - 내용 : 최대 1000자
        - 글쓴이 : 최대 50자
    - '수정' 버튼을 클릭하면 사용자가 입력한 글이 수정되고, 글 목록 페이지로 전환합니다. 글 목록 페이지

글 목록 페이지 : /list
    - 게시물의 개수와 상관없이 한 페이지에 등록된 모든 게시물을 표시합니다.
    - 가장 최근에 쓴 글을, 목록의 가장 상단에 표시합니다.
    - 목록에서 표시해야 할 컬럼의 종류
        - 제목
        - 글쓴이
        - 글 쓴 시각
        - 읽은 회수
    - 글의 제목을 클릭하면 글 읽기 페이지로 전환합니다.
    - 목록의 윗 편에 '글쓰기' 버튼이 있어서, 클릭하면 글 쓰기 페이지로 전환합니다

글 읽기 페이지 : /view
    - 페이지를 호출 할 때 전달한 아이디 값은 PathVariable, RequestParam 어느 것이나 사용하여도 좋습니다.
    - 표시해야 할 컬럼
        - 제목
        - 내용
        - 글쓴이
        - 글 쓴 시각
        - 읽은 회수
    - 이 페이지에 진입할 때, 해당 글의 읽은 회수를 1 증가시킵니다.
    - 페이지의 하단에 '목록' 버튼을 두어, 클릭하면 글 목록 페이지로 전환합니다.

제약 사항
데이터베이스를 access 하기 위해서 mybatis나 JPA 가운데 하나를 선택해서 사용하세요.
DBMS는 아무 것이나 사용해도 좋습니다. 다만 평가를 위해서 외부에서 접속해야 함으로, 고정 주소로 외부에서 접속이 가능한 데이터베이스를 사용하거나, 내장 데이터베이스(예를 들어 h2database)을 사용하세요.


평가 방법

기능 구현 : 30점 ( 4개 구현 30, 3개구현 25, 2개구현 20, 1개구현 15)
디자인 : 20점 
소스 코드 : 20점 ( 변수명, 클래스명, 스프링프레임워크개발패턴준수, 주석 )
제출방법 준수 : 20점

참고사항
[기능을 추가적으로 구현하거나 디자인 퀄리티를 좋게 할수록 가점을 부여하겠습니다.]

제출 방법
1. 소스파일
2. 실행 동영상 파일

두개 파일을 zip으로 압축해서 클래스룸으로 제출하세요.
.jar 로 패키징 하세요. 

8080 포트를 사용하세요.

교사가 zip 파일을 풀고 해당 프로젝트를 intellij에서 open 한 뒤, 아무런 작업 없이 프로그램을 수행할 수 있도록 제출하세요.
데이터베이스 주소를 고쳐야 하는 등 별도의 작업이 필요하지 않게 제출하세요.

교사가 해당 프로그램을 수행시킨 뒤, 웹 브라우저에서 http://localhost:8080/list 페이지를 바로 띄울 수 있게 제출하세요.
